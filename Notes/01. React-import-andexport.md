# JavaScript / React Exports & Imports – Quick Revision

## 1. Default Export

Use when a file exports **one main thing**.

### Export

```javascript
function Dashboard() {
  return <div>Dashboard</div>;
}

export default Dashboard;
```

### Import

```javascript
import Dashboard from "./Dashboard";
```

### Key Points

- No `{}` used during import
- You can rename it while importing

```javascript
import MyComponent from "./Dashboard";
```

---

# 2. Named Export

Use when a file exports **multiple things**.

### Export

```javascript
export const ThemeContext = createContext();

export function ThemeProvider({ children }) {
  return <ThemeContext.Provider>{children}</ThemeContext.Provider>;
}
```

### Import

```javascript
import { ThemeContext, ThemeProvider } from "./ThemeContext";
```

### Key Points

- Must use `{}` during import
- Import name **must match export name**

---

# 3. Multiple Named Exports

### Export

```javascript
export function Button() {}
export function Card() {}
export function Modal() {}
```

### Import

```javascript
import { Button, Card, Modal } from "./components";
```

---

# 4. Mixing Default + Named Export

### Export

```javascript
export const ThemeContext = createContext();

export function ThemeProvider({ children }) {}

export default function ThemeToggle() {}
```

### Import

```javascript
import ThemeToggle, { ThemeContext, ThemeProvider } from "./theme";
```

### Rule

| Type    | Import Syntax                |
| ------- | ---------------------------- |
| Default | `import X from "./file"`     |
| Named   | `import { X } from "./file"` |

---

# 5. Quick Mental Rule

If you see:

```
export default
```

➡ Import **without `{}`**

If you see:

```
export const
export function
export class
```

➡ Import **with `{}`**

---

# 6. Context Pattern (Common in React)

### Context File

```javascript
import { createContext } from "react";

export const TestContext = createContext();

export function TestProvider({ children }) {
  const value = { name: "Mayank" };

  return <TestContext.Provider value={value}>{children}</TestContext.Provider>;
}
```

### Import

```javascript
import { TestContext, TestProvider } from "./TestContext";
```

---

# 7. Custom Hook Pattern

### Hook File

```javascript
import { useContext } from "react";
import { TestContext } from "../context/TestContext";

export function useTest() {
  const context = useContext(TestContext);

  if (!context) {
    throw new Error("useTest must be used inside TestProvider");
  }

  return context;
}
```

### Import

```javascript
import { useTest } from "../hooks/useTest";
```

---

# Final Summary

| Export Type    | Syntax                     | Import                                |
| -------------- | -------------------------- | ------------------------------------- |
| Default Export | `export default Component` | `import Component from "./file"`      |
| Named Export   | `export const X`           | `import { X } from "./file"`          |
| Mixed          | Both together              | `import Default, { X } from "./file"` |
